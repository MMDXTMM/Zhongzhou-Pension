## 第3页：项目背景与需求分析

### 行业背景

随着我国人口老龄化程度不断加深，养老服务需求持续增长。传统养老管理模式面临诸多挑战：

- **管理效率低下**：人工登记、纸质档案，信息查询困难，容易出错
- **服务质量难以保障**：护理流程不规范，服务标准不统一
- **资源调度困难**：床位管理、人员排班、物资分配依赖人工经验
- **安全隐患突出**：老人健康状况难以及时掌握，意外事件响应滞后
- **家属沟通不畅**：家属无法及时了解老人在院情况，信息不对称

### 项目目标

打造一套集**健康管理、护理服务、运营管理、智能决策**于一体的智慧养老管理平台，实现养老机构的数字化、智能化、精细化管理，全面提升养老服务质量和运营效率。

## 第4页：系统整体架构

**前端技术栈：**
- Vue.js 2.6 + Element UI 组件库

- 动态路由与权限控制

**后端技术栈：**
- Spring Boot 3.5 + Java 21
- MyBatis Plus 持久层框架

## 第5页：核心功能模块 - 入退管理

### 健康评估
- 建立完善的老人健康档案，包括基本信息、既往病史、过敏史等
- 支持多维度健康评估：健康等级、认知等级、行动能力、自理能力
- 评估流程标准化，确保评估结果客观准确
- 评估数据与入住、护理模块联动，为后续服务提供数据支撑

### 入住办理
- 全流程化入住管理，从申请到入住一站式完成
- 关联健康评估结果，自动推荐护理等级和护理计划
- 智能床位分配，支持楼栋、房间、床位三级管理
- 合同管理与费用结算，押金、月费用清晰透明
- 紧急联系人信息管理，确保紧急情况及时沟通

### 退住管理
- 规范的退住申请与审核流程
- 支持多种退住原因：回家养老、转院治疗、家属接回等
- 费用自动结算，结算金额、退款金额清晰明了
- 退住记录完整留存，便于追溯和统计分析

## 第6页：核心功能模块 - 养老护理

### 护理项目管理
- 丰富的护理项目库，涵盖生活照料、基础护理、康复护理等多个类别
- 每个项目包含名称、单位、价格、护理要求等详细信息
- 支持项目的启用/禁用状态管理，灵活调整服务内容
- 排序号功能，支持自定义展示顺序

### 护理计划管理
- 根据老人健康状况制定个性化护理计划
- 护理计划与护理项目关联，明确服务内容和频次
- 支持护理计划的版本管理，方便调整和优化
- 计划执行跟踪，确保护理服务落实到位

### 护理等级管理
- 科学的护理等级划分，从自理到特护多级分类
- 每个等级对应不同的护理费用和服务标准
- 护理等级与护理计划关联，实现标准化服务
- 支持等级动态调整，根据老人健康状况变化及时更新

### 来访管理
- 规范的访客登记与审批流程
- 访客信息完整记录：姓名、电话、身份证号、与老人关系
- 来访预约管理，合理安排来访时间
- 签到/签退登记，精确掌握访客在院时间
- 来访数据统计分析，优化访客管理策略

## 第11页：系统安全保障

### 数据安全
- **密码加密**：用户密码采用BCrypt加密存储，不可逆
- **传输加密**：支持HTTPS协议，数据传输加密
- **数据备份**：支持数据库定期备份，防止数据丢失
- **权限控制**：细粒度的权限管理，防止越权访问

### 网络安全
- **防火墙**：网络边界防护，阻止非法访问
- **访问控制**：IP黑白名单，限制访问来源
- **防SQL注入**：参数化查询，防止SQL注入攻击
- **防XSS攻击**：输入输出过滤，防止跨站脚本攻击

### 应用安全
- **登录认证**：用户名密码 + 验证码双因子认证
- **会话管理**：Token机制，会话超时自动失效
- **操作审计**：所有操作留痕，支持审计追溯
- **敏感数据脱敏**：敏感信息展示时自动脱敏

### 运维安全
- **日志监控**：完整的操作日志和系统日志
- **异常告警**：系统异常及时通知管理员
- **版本管理**：代码版本控制，便于回滚
- **灾备方案**：数据备份与恢复机制完善
